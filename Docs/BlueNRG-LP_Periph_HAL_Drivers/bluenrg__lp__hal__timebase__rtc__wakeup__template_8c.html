Detailed Description

HAL time base based on the hardware RTC_WAKEUP Template.

Author
RF Application Team This file overrides the native HAL time base functions (defined as weak) to use the RTC WAKEUP for the time base generation:
  • Intializes the RTC peripheral and configures the wakeup timer to be incremented each 1ms
  • The wakeup feature is configured to assert an interrupt each 1ms
  • HAL_IncTick is called inside the HAL_RTCEx_WakeUpTimerEventCallback
  • HSE (default), LSE or LSI can be selected as RTC clock source 
     ==============================================================================
                       ##### How to use this driver #####
 ==============================================================================
   [..]
   This file must be copied to the application folder and modified as follows:
   (#) Rename it to 'bluenrg_lp_hal_timebase_rtc_wakeup.c'
   (#) Add this file and the RTC HAL drivers to your project and uncomment
      HAL_RTC_MODULE_ENABLED define in bluenrg_lp_hal_conf.h 

   [..]
   (@) HAL RTC alarm and HAL RTC wakeup drivers can’t be used with low power modes:
       The wake up capability of the RTC may be intrusive in case of prior low power mode
       configuration requiring different wake up sources.
       Application/Example behavior is no more guaranteed 
   (@) The bluenrg_lp_hal_timebase_tim use is recommended for the Applications/Examples
         requiring low power modes
